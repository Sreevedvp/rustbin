About RustBin Sandbox Tech

An off-line technical document outlining how this compiler playground sandboxes, compiles, and micro-benchmarks code.

⚙️ How Local Subprocess Runtimes Work

When you click Run inside the playrooms, the backend engine writes your text inputs to a clean temporary file within the temp_runs folder, completely sandboxed from standard workspace directories. Subprocess commands are called locally:

  • Rust Compiler Sandbox: Compiles code using the local rustc toolchain, builds it, runs the resulting binary with stdout/stderr pipes, and executes in $< 200$ms.
  • JavaScript Runtime Sandbox: Spawns a local node command to interpret Javascript files on your machine's Node.js compiler environment offline.

⏱️ High-Precision V8 and Release Benchmarking

To bypass random background scheduling noise from the operating system, RustBin uses statistical profiling:

1. CPU Warmup Passes: Runs the function loop 15-20 times beforehand. This heats up the CPU cache and triggers Node's V8 JIT TurboFan compiler to compile and optimize Javascript hot paths before measurements start.

2. Compiler Opt-Outs: Applies std::hint::black_box statements in Rust to keep the compiler from stripping out dead benchmark loops.

3. 95% Confidence Intervals: Runs the loops 200 times. Runtimes are mathematically sorted to extract the Mean ($\mu$) and Standard Deviation ($\sigma$), and standard error is solved to establish a 95% Confidence Margin of Error (MoE):

$MoE = \pm 1.96 \times \frac{\sigma}{\sqrt{N}}$

🔍 Heuristic AST Optimization Scanners

Our local optimizer performs structural scans without internet access or AI dependencies:

  • Clippy warnings: Copies your Rust file to a local dummy package and compiles `cargo clippy` to extract compiler-driven lint diagnostics natively.
  • V8 and Rust AST Heuristics: Scans the code structures for O(N^2) quadratic lookups, redundant heap allocations, and loop vector capacity grow bottlenecks, outputting clear optimized comparisons.
